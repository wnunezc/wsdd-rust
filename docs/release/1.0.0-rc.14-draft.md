# Release Draft — WSDD 1.0.0-rc.14

## Status

Draft release notes prepared locally. Do not publish yet.

## Recommended package

- **Primary asset:** `wsdd-1.0.0.14-x86_64.msi`
- **SHA256:** `8BDB802F90277D3BAD495E622E2C488A2F9BDD299FB43EF280E0490643FFCD8B`
- **Target audience:** Windows 10 / 11 users running WSDD with administrator privileges
- **Distribution channel:** Private or draft GitHub Release until publication is explicitly approved

## Highlights

- `WP-04` closed locally in `RC14` after stabilizing background jobs and UI non-blocking behavior
- prerequisites reordered to `PowerShell 7.5+ -> Chocolatey -> MKCert -> Docker`
- PowerShell bootstrap no longer depends on Chocolatey
- base Docker environment now waits for real readiness instead of fixed 15-second sleeps
- first-run Loader behavior restored and action buttons enlarged for visibility
- log output standardized with level icons and preserved color coding
- browser URLs now open without console flicker

## Validation

- `cargo fmt --all`
- `cargo check --workspace`
- `cargo clippy --workspace -- -D warnings`
- `cargo wix-msi --nocapture`
- `cargo test --workspace` compiles but does not execute in this environment because Windows requires elevation (`os error 740`)

## Notes for future developers

`WP-04` is considered closed by user decision in `RC14`, but a few non-blocking technical follow-ups remain documented for future work:

- there is no formal `queued` state yet in the background job model
- a few utility/background helpers still use local threads outside the central coordinator:
  - folder picker in `src/ui/add_project.rs`
  - process/log bridge helpers in handlers
  - internal reader threads in `ps_script.rs`
- these items were intentionally not expanded further in this phase to avoid reopening `WP-04`

## Suggested release summary

WSDD `1.0.0-rc.14` closes `WP-04` and stabilizes the job execution strategy introduced after `RC13`. This build makes prerequisites more reliable on clean systems, removes fixed readiness sleeps from the base Docker setup, restores the intended first-run Loader behavior, improves log readability, and eliminates browser-launch console flicker.
